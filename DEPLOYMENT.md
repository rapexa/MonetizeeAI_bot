# راهنمای Deploy لندینگ پیج

## فایل‌های مورد نیاز:

1. **landing.html** - فایل اصلی لندینگ پیج (بدون محدودیت دسترسی)
2. **index.html** - صفحه محدودیت دسترسی (برای بقیه صفحات)
3. **.htaccess** - فایل تنظیمات Apache برای مسیریابی و محدودیت دسترسی

## مراحل Deploy:

### 1. آپلود فایل‌ها به سرور:

فایل‌های زیر را به root directory سرور (یا پوشه public_html) آپلود کنید:
- `landing.html`
- `.htaccess`

### 2. تنظیمات سرور:

#### برای Apache:
- فایل `.htaccess` به صورت خودکار کار می‌کند
- مطمئن شوید که `mod_rewrite` فعال است

#### برای Nginx:
اگر از Nginx استفاده می‌کنید، باید تنظیمات زیر را به فایل config اضافه کنید:

```nginx
location /landing {
    try_files $uri $uri/ /landing.html;
}

location = /landing.html {
    # Serve the landing page
}
```

#### برای سرورهای دیگر:
- می‌توانید مستقیماً از `landing.html` استفاده کنید
- یا یک redirect ساده در تنظیمات سرور ایجاد کنید

### 3. تست:

بعد از آپلود، آدرس‌های زیر را تست کنید:

**صفحه لندینگ (بدون محدودیت):**
- `http://sianmarketing.com/landing` ✅ باید لندینگ پیج نمایش داده شود
- `http://sianmarketing.com/landing/` ✅ باید لندینگ پیج نمایش داده شود
- `http://sianmarketing.com/landing.html` ✅ باید لندینگ پیج نمایش داده شود

**صفحات دیگر (با محدودیت):**
- `http://sianmarketing.com/` ❌ باید صفحه محدودیت نمایش داده شود
- `http://sianmarketing.com/any-page` ❌ باید صفحه محدودیت نمایش داده شود

## نحوه کار:

سیستم به این صورت کار می‌کند:
- **صفحه `/landing`**: بدون محدودیت، مستقیماً لندینگ پیج نمایش داده می‌شود
- **بقیه صفحات**: صفحه محدودیت (`index.html`) نمایش داده می‌شود که کاربر را به ربات تلگرام هدایت می‌کند

## نکات مهم:

1. **فایل .htaccess**: مطمئن شوید که فایل `.htaccess` با نقطه شروع می‌شود (نه `htaccess`)
2. **Permissions**: فایل‌ها باید قابل خواندن باشند (chmod 644)
3. **Cache**: ممکن است نیاز باشد cache مرورگر را پاک کنید
4. **mod_rewrite**: باید در Apache فعال باشد

## پشتیبانی:

اگر مشکلی پیش آمد، بررسی کنید:
- آیا `mod_rewrite` در Apache فعال است؟
- آیا فایل `.htaccess` درست آپلود شده است؟
- آیا مسیر فایل‌ها درست است؟
